# プロジェクトメモ

## UI/UX について

UI/UX に関しては DESIGN.md を参照してください。

## 概要

- このリポジトリは Pokemon Champions 向けのダメージ計算ツールです。
- アプリは Next.js App Router と TypeScript で実装しています。
- 配備先は Vercel を想定しています。

## 構成

- `app/`
  - Next.js のルーティングと Route Handler を置く
- `components/`
  - クライアント側の UI コンポーネントを置く
- `src/`
  - 再利用するドメインロジックを置く
  - ダメージ計算
  - チャンピオンズ仕様の実数値計算
  - PokeAPI 取得とローカルキャッシュ

## ドメイン前提

- バトルレベルは 50 固定です。
- IV は 31 固定です。
- SP は現状このリポジトリ内で次の前提で扱っています。
  - 合計上限は 66
  - 1能力あたりの上限は 32
- 性格補正を適用します。
- 日本語のポケモン名は、PokeAPI を元にしたローカルキャッシュで解決します。

## データの扱い

- ポケモンデータの元情報は PokeAPI です。
- ローカルキャッシュは `.cache/pokeapi/` に保存します。
- `.cache/` は開発補助用なのでコミットしません。
- PokeAPI から導ける情報は、大きな手書き辞書を増やさずに対応する方針です。

## 開発ルール

- `src/calculator.ts`、`src/stats.ts`、`src/types.ts` はできるだけ再利用します。
- UI の責務は `components/`、ルーティングの責務は `app/` に寄せます。
- 計算ロジックを React コンポーネントに直接書き込まないようにします。
- 新しいゲーム要素を追加するときは、まずドメイン層を拡張し、その後 UI をつなぎます。
- `components/calculator-app.tsx` は画面全体の状態管理と取得処理を担当し、表示部分は `components/calculator/` 配下の小さなコンポーネントへ分割します。
- このリポジトリで新しく追加するファイル名は、フレームワークや慣習上の固定名を除いて camelCase を使います。
- UI を大きくする場合は、表示責務ごとに `PokemonPanel`、`MovePanel`、`ResultPanel` のような単位で分割してから見た目を調整します。
- 新しい UI 部品を追加するときは、まず既存コンポーネントへ責務を寄せるかを検討し、肥大化する場合だけ新しいファイルを作ります。

## 確認項目

- UI ルールやデザイン関連の変更後は `npm run design:check` を実行します。
- `DESIGN.md` と contracts を更新した場合は `npm run design:drift` も実行します。
- まとまった変更の後は `npm run build` を実行します。
- キャッシュ生成ロジックを触った場合は `npm run refresh:pokeapi-cache` も確認します。

# GUI Requirements

Pokemon Champions Damage Calc の GUI 要件整理です。  
このファイルの目的は、見た目の議論ではなく「この GUI が何を解決するか」「何を先に実装するか」「どこまでを今回の範囲にするか」を固定することです。

`DESIGN.md` は見た目の原則、`docs/gui-requirements.md` は画面の要件と機能スコープを扱います。

## 1. この文書の使い方

- GUI 改修を始める前に、この文書で対象機能が `Must` か `Should` かを確認する
- 新しい画面要素を追加する前に、対応するユーザーストーリーと受け入れ条件を更新する
- 実装タスクは、この文書の `Must` と `Should` をもとに backlog 化する

## 2. 背景

このアプリは CLI 的な発想で積み上がった部分があり、現在の GUI は次の状態です。

- ポケモン検索、性格、SP 配分、ダメージ計算はできる
- 技条件は手入力前提で、GUI としては未完成
- 結果表示は最低限あるが、対戦ツールとしての判断補助はまだ弱い
- 将来追加したい機能と、今回の GUI の完成ラインがまだ混ざっている

そのため、まず GUI の目的と優先順位を明文化する。

## 3. プロダクトゴール

この GUI の第一目的は、対戦中または育成検討中のユーザーが、短時間でダメージ感覚を確認できることです。

この GUI は次を実現する。

- 攻撃側と防御側の条件を迷わず入力できる
- Champions 仕様の前提で実数値とダメージを計算できる
- 結果を「数値を見る」だけでなく「判断に使える」形で表示できる

この GUI は、Pokémon battle simulator の完全再現を目指さない。

## 4. 想定ユーザー

### メインユーザー

- Pokemon Champions の対戦や育成を行うプレイヤー
- 実数値やダメージ感覚を GUI で素早く確認したい人

### サブユーザー

- データ検証や仕様確認をしたい開発者
- Champions 仕様の計算結果をざっと確認したい人

## 5. 成功条件

次の状態を満たせば、GUI の最小価値は成立しているとみなす。

- 初見ユーザーでも、攻撃側・防御側・技条件を見て役割を理解できる
- ポケモン指定から結果表示までを 1 画面で完結できる
- SP 制約や入力エラーが分かりやすい
- 少なくとも「この条件ならどの程度入るか」を数十秒で確認できる

## 6. 優先順位の判断基準

機能追加の優先順位は、次の 4 点で判断する。

1. ユーザー価値が高いか
2. 早くフィードバックを得る価値があるか
3. 実装コストに対して効果が高いか
4. 他機能の前提になる依存関係があるか

## 7. ユーザーストーリー

### US-01 条件を入力してダメージを知りたい

As a 対戦プレイヤー, I want 攻撃側・防御側・技条件を入力したい, so that ダメージレンジをすぐ確認できる。

### US-02 日本語名でポケモンを探したい

As a 日本語話者のユーザー, I want 日本語名でポケモンを検索したい, so that PokeAPI 名を知らなくても使える。

### US-03 Champions 仕様で実数値を確認したい

As a 育成検討中のユーザー, I want 性格と SP 配分から実数値を見たい, so that 対戦用の調整を確認できる。

### US-04 入力ミスを早く知りたい

As a ユーザー, I want 不正入力や上限超過をすぐ知りたい, so that 間違った条件で計算しないですむ。

### US-05 結果を判断に使いたい

As a 対戦プレイヤー, I want ダメージ結果を割合や確定情報つきで見たい, so that 次の行動を判断しやすくなる。

## 8. 受け入れ条件

### AC-01 ポケモン入力

- 攻撃側と防御側は別々にポケモンを指定できる
- 日本語名と英字名のどちらでも入力できる
- ポケモン取得に失敗した場合は、ユーザーに分かる文言を表示する

### AC-02 実数値入力補助

- 性格を選べる
- 各能力の SP を入力できる
- 1 能力あたり上限と合計上限を GUI 上で確認できる
- 上限超過時はエラーとして分かる

### AC-03 技条件入力

- 少なくとも分類、威力、タイプを指定できる
- 現段階では手入力でもよいが、将来的に技名からの自動反映へ拡張可能な構造にする

### AC-04 結果表示

- ダメージ最小値と最大値を表示する
- HP 比率を表示する
- タイプ相性と STAB の有無を表示する
- 攻撃実数値と防御実数値を表示する
- 確定一発かどうかを表示する

### AC-05 エラーと状態

- ポケモン未入力時に取得ボタンを押した場合、入力を促す
- SP 上限超過時に結果を出さない
- 技威力が不正な場合はエラーを表示する
- API 失敗時に沈黙しない

## 9. 画面構成要件

### Must

- Hero または画面概要
- 攻撃側ポケモンパネル
- 防御側ポケモンパネル
- 技条件パネル
- 結果パネル

### Should

- 入力項目ごとの補助文
- エラーの視覚的強調
- モバイルで 1 カラムに崩れるレスポンシブ対応

### Nice to have

- 候補サジェスト
- 乱数分布の視覚化
- 結果のコピー

## 10. 機能優先度

### Must

- ポケモン検索
- 日本語名対応
- 性格選択
- SP 入力
- Champions 実数値計算
- 技の分類、威力、タイプ入力
- ダメージ最小最大表示
- ダメージ割合表示
- STAB / タイプ相性表示
- 入力エラー表示

### Should

- 技名検索からの自動反映
- 結果表示の読みやすい整理
- API エラーの分類表示
- 入力途中でも分かりやすいバリデーション

### Could

- 検索候補サジェスト
- 技履歴
- 条件の共有 URL
- ロール別プリセット

## 11. 今回のスコープ外

この段階では、次は扱わない。

- 特性
- 持ち物
- 天候
- フィールド
- 壁
- 急所
- 乱数分布の高度な分析
- Champions 内定ポケモンのみへの限定
- 本番向けキャッシュ戦略

## 14. この文書を更新するタイミング

次のどれかをやる前に更新する。
